This tool generates look-alike domains and scores them based on similarity.

### Domain Generation Techniques

| Technique | Description | Example |
|-----------|-------------|---------|
| **Omission** | Removes single characters | `gogle.com` |
| **Repetition** | Duplicates characters | `gooogle.com` |
| **Transposition** | Swaps adjacent characters | `googel.com` |
| **Hyphenation** | Inserts hyphens | `goo-gle.com` |
| **Vowel Swapping** | Replaces vowels with other vowels | `googla.com` |
| **Keyboard Typos** | Adjacent key substitutions (QWERTY, AZERTY, QWERTZ) | `gpogle.com` |
| **Homoglyphs** | Visually similar characters (accents, diacritics) | `gĞ¾Ğ¾gle.com` |
| **Cyrillic/Greek** | Full script substitution | `gÎ¿Î¿gle.com` |
| **Combosquatting** | Adds common prefixes/suffixes | `login-google.com` |
| **TLD Variations** | Alternative top-level domains | `google.co`, `google.cm` |

### Similarity Scoring

Each generated domain is scored based on:
- **Apex Similarity (50%)** - Levenshtein distance comparison
- **TLD Similarity (30%)** - How confusable the TLD is with the original
- **Method Risk Weight (20%)** - Deceptiveness of the generation technique

## Installation

1. Clone the repository
```shell
git clone https://github.com/RIZZZIOM/phishfolio/domain-doppelganger.git
cd domain-doppelganger
```

2. Create a virtual environment and install requirements
```shell
python -m venv venv
source venv/bin/activate # linux
.\venv\Scripts\activate # windows

pip install -r requirements.txt
```

## Output

The tool generates:
- **Terminal Display** - Top 50 highest-risk domains in a formatted table
- **JSON File** - Complete results saved to `doppelganger_<domain>_<timestamp>.json`

### JSON Output Structure

```json
{
  "scan_info": {
    "original_domain": "example.com",
    "scan_timestamp": "2026-01-24T12:00:00",
    "total_domains_generated": 10000
  },
  "domains": [
    {
      "domain": "examp1e.com",
      "punycode": "examp1e.com",
      "method": "homoglyphs",
      "similarity_score": 0.95,
      "apex_similarity": 0.875,
      "tld_similarity": 1.0,
      "method_weight": 1.0
    }
  ]
}
```

## How It Works

1. **Input Validation** - Validates domain format and characters (ASCII & IDN support)
2. **Domain Parsing** - Extracts apex domain, subdomain, and TLD
3. **Variation Generation** - Applies multiple typosquatting techniques
4. **TLD Expansion** - Combines variations with common TLDs
5. **Scoring** - Calculates risk scores based on similarity metrics
6. **Output** - Displays top results and saves complete data to JSON

### Use Cases

- **Brand Protection** - Monitor for domains impersonating your brand
- **Phishing Detection** - Identify potential phishing infrastructure
- **Security Research** - Study typosquatting patterns
- **Threat Intelligence** - Generate IOCs for blocking
